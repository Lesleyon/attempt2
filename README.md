Метод simulate
Сложность: O(n^3)
Данный метод получает данные о двух армиях, а именно игрока и компьютера, после чего собирает их в один общий массив очереди 
Создание очереди происоходит пока один из массивов юнитов из армий не станет равным 0
После создания очереди если была найдена цель и юнит до неё дошёл и добавляется, и информация об атаки добавляется в лог
если цель умерла после атаки то она удаляется из очереди и оригинального массива армии
В конце дополнительное удаление всех умерших юнитов в раунде ака фейлсейф в случае если какая-либо информация не была удалена в прошлом пункте

Метод getSuitableUnits
Сложность O(n^2/3c), где с - количество проходок по столбца
Код находит цель, которую он хочет атаковать
Сначала определяем направление для армии
1 - атака правой армии
-1 - атака левой армии
Затем происходит переборка всех юнитов по рядам и проверка на блокировку для атаки левой армии (атакуемой) закрывают ли юнита юнит спереди и сбоку от него
то же самое для правой атакуемой армии
Затем подходящие юниты добавляются в список юнитов подходящих для атаки

Метод getTargetPath
Сложность O(W^2*H^2), где W-21 и H - 27
Код - создан код по принципу жадного алгоритма для движения используемый в игровых движках
Изначально определяются координаты атакующего, цели и поля.
Создаётся путь и координаты атакующего и цели, а также изначально пустой занятых клеток, который заполняется сразу клетками, на которых появляются юниты обоих армий
После чего инициализируются x и y для определения текущей позиции атакующего и начинается цикл для движения
Определяется направление движения, относительно текущей координаты и координаты цели и возможность двигаться дальше по циклам
1. Движение по диагонали
	-оба направления одинаковы
	-инициализируются временные переменные равные координате свободной клетки, формируемые из текущих координат и координат движения	
	-юнит переходит на клетку
	-проверка на то, что вюнит не вышел за координаты поля	или клетка уже занята
	-текущие координаты юнита добавляются в его путь
	-текущие координаты юнита становятся координатами клетки
	-флаг сохраняет значение true
2.Движение по горизонтали
	-направление вертикали равно 0
	-инициализируются временные переменные равные координате свободной клетки, формируемые из текущей координаты  и координаты движения по горизонтали, текущей координаты вертикали	
	-юнит переходит на клетку
	-проверка на то, что вюнит не вышел за координаты поля	или клетка уже занята
	-текущие координаты юнита добавляются в его путь
	-текущие координаты юнита становятся координатами клетки
	-флаг сохраняет значение true
3.Движение по вертикали
	-направление горизонтали равно 0
	-инициализируются временные переменные равные координате свободной клетки, формируемые из текущей координаты  и координаты движения по вертикали, текущей координаты горизонтали
	-юнит переходит на клетку
	-проверка на то, что вюнит не вышел за координаты поля	или клетка уже занята
	-текущие координаты юнита добавляются в его путь
	-текущие координаты юнита становятся координатами клетки
	-флаг сохраняет значение true
4. Если вообще движения никакого нет
	-выход их цикла
Когда юнит доходит до цели, то координаты цели добавляются к пути

затем начинется цикл с прохода по unitList
инициализируется цена, тип и счётчик
